# 篇章壹：日报周报篇（生财有术搜索引擎）

我做这个前后花了快十天时间，当然前期的工具测试花费时间居多，现在基本已经完全跑通自动化了。

一切都来源于上周亦仁所发的周报，我当时想着简单评论一下RPA或许也可以实现。

![](img/9c999e9b4d498421956ef7aa6994254d.png)

但事实上，当我真正开始着手的时候，我发现知识星球和生财有术的搜索引擎，全部都限制内容抓取。

所以我在第一步，批量获取帖子信息（标题和链接），就卡壳了三五天时间。

这其中我尝试了用Uibot（我最擅长却熟悉）和Automa（谷歌浏览器插件）

结果效果并不理想，用是能用，但是Uibot耗时长达一个小时，虽然可以做到完全自动化（甚至我把ChatGPT的prompt都调试了几十遍得出最终版）;然后那个谷歌插件Automa太薄弱了，我代码弄得过多，导致插件非常卡顿，当然速度也算是比较快，就是最后还得加入人工去发给ChatGPT（因为浏览器插件不如桌面软件那样可以完全控制本地文件）

仅仅是为了准确地获取每一个帖子的链接，我都花费了一整周的时间，因为知识星球和生财有术的网页版元素全部都做了反爬处理，我只能完全模拟人工去慢慢操作，于是RPA搞得又慢又卡（我总共弄了快两万行代码），而且还容易出错。（虽然最后文档版本的基本没问题了，但我感觉还是略有瑕疵。）

一直想着完蛋，这没办法交差了，难道是我技术不行？

本来我不想用影刀的，因为这个软件免费版本看不到代码，而且还没办法导出共享给其他电脑，所以我就一直觉得这是一个免费阉割版的RPA。

直到我发现了影刀的高级用法，爆款循环相似元素和模拟键盘点击（我以前一直以为影刀没这个功能），进而我还研究出如何查看影刀脚本的源代码，以及共享复制给其他账号其他电脑（出于可能侵犯影刀商业利益原因，这里就不公开阐述了），于是我发现影刀还是比较适合某些复杂多流程防反爬项目当中的。

这里非常感谢阿盛@邹培盛跟我关于影刀的用法交流。

### 一，自动化步骤配置思路

最简化是两个步骤，这是站在岸上看别人游泳时觉着的简单：不就两条腿往后蹬吗？

![](img/d9dcef2d2fad509abbae1aac7704fd7a.png)

但真正执行起来,RPA快超过2W行代码，大概两千个步骤，因为要绕过帖子反爬，没办法运用循环简化。

这就是做起来，真感觉跟刚学游泳的时候一样，虽然我在岸上练习了标准动作很久（UIBOT我确实很熟，都可以闭眼写基本流程代码了），但是等下到水里面只感觉胸闷气短，两条腿有力气使不上劲，然后就老师呛水（真正项目市场，我反复测试了各种方法和方向，不算调试时间，方案就整了七八套去部署）

简单提一下，RPA的核心并不是代码，而是流程设计，速成你人工要怎么做的SOP，然后就需要交给机器人来完成，当然为了批量自动化的效率考量，这里面还有新的迭代升级，可能要借助一些插件什么的。

换句话说，两个都懂RPA的圈友，可能针对同一个项目，设计出来的是完全不同的流程（可能借用的工具也不同）

这个想法是我在看其他圈友设计公众号RPA批量自动化的思路冒出来的，跟我的确实大不相同。

![](img/d9dd61bac79cab108195ab6ba0a69299.png)

UIBOT因为缺乏影刀的相似元素循环获取，所以就导致代码比较多，影刀500行代码解决（内置二十次循环*8倍），Uibot是花了5091行代码解决（外置循环因为反爬用不了）。

当然也可能是我学艺不精，没徒手写自定义代码，或者说找到更高阶的用法，UIBOT我确实是花了三天部署，而影刀就只花了一天。（有效制作流程时间也就三个小时，剩下的就是调试，因为我写代码和流程总是考虑不周到，犯的很多错误就需要在测试中一点点纠正）。

### 二，具体实操遇到各种问题

本来以为操作非常简单，因为逻辑就两个步骤：获取帖子信息+发给ChatGPT

但是在第一步我卡了三天，因为生财有术搜索引擎和知识星球都没办法抓取帖子详细信息

RPA无法固定元素获取，我换了Automa浏览器自动化插件获取，但是元素太多非常容易卡顿

最后还是用Uibot完成，只能抓取三部分数据，比较多信息叠在一起

又导致ChatGPT没办法用正则表达式抽取详细信息

因此目前仅是一个粗略版本。

更新：所有问题已解决，只是代码比较多，最完美版本是VPS+RPA。我的方案是用腾讯云VPS，RPA机器人放在VPS里面，24H运行不用管，定制化发消息弄个单独微信即可。
