## 六）怎么解决因为上下文 token 过长，大模型遗忘了之前的对话？怎么解决因为提示词占用 token 过长，大模型开始胡言乱语？

1.  问题诊断和分析能力

回答框架：

STAR 模型（ Situation （情境）， Task （任务）， Action （行动）， 和 Result （结果））

参考话术：

其实您这是 2 个问题，我把他拆开 2 个问题：

问题 1: 怎么解决因为上下文 token 过长，大模型遗忘了之前的对话？

问题 2: 怎么解决因为提示词占用 token 过长，大模型开始胡言乱语？

针对第一个问题：

在解决token这个方面的话 目前的常规做法就是压缩自己的表达结构 从语义的方面尽量压缩输入和输出内容 从专业角度就是使用信息熵更高的表达方式

总之一句话：在用户端，压缩Token的最好办法就是让自己的表达精炼，不说废话

针对第二个问题：

S（Situation）情境：

在使用大型模型时，有时提示词的长度会导致占用太多 token，从而使模型开始产生无关的、胡言乱语的回复。

T（Task）任务：

我的任务是找到方法来解决提示词占用 token 过长的问题，确保模型仍能提供有意义的回复。

A（Action）行动：

首先，我会对提示词进行精简，保持关键信息，同时减少冗余或不必要的内容。这可以有效降低 token 使用。

其次，我会使用截断和摘要技术，将过长的提示词进行截断或生成简短摘要，以确保其适应模型的限制。

最后，我会尝试不同的模型配置，包括模型大小和 token 上限，以找到最适合特定任务的配置，从而平衡提示词长度和模型性能。

R（Result）结果：

通过这些方法，我们能够解决因为提示词占用 token 过长而导致模型胡言乱语的问题，确保模型仍能提供有意义的回复。

*   如果加入公司，我想更快的熟悉工作、贡献自己价值，想了解一下，我们有明确的工作流程吗？（可问直属领导）

*   如果加入公司，在您看来，您希望我在短期内解决哪些问题，长期解决什么问题？（可问直属领导）

*   您觉得这个岗位最具挑战性的地方在哪里？我想在加入公司前能提前准备，加入后，能快速的适应（可问直属领导）

*   如果在团队中有能较好的贡献，我也想挑战更高的目标，那这个岗位的晋升路径是什么样子的？（可问直属领导）

*   如果能加入咱们团队，试用期考核您最看重哪一点？我不想让您失望（可问直属领导）

*   咱们这次沟通下来，您对我哪一点印象最深刻呢？（可问直属领导）

### 3、判断领导风格和水平

工作风格：直属领导有哪些特定的工作习惯？

角度一：平时和您配合工作时，您有哪些特定的工作习惯吗？

角度二：向您汇报工作时，您更布望我用哪种方式？多久汇报-次比较好？

角度三：您在工作中比较注重什么？对下属的工作，最看重什么？

理念态度：直属领导对待工作的态度如何？

角度一：您一般用哪些方式关注团队业绩情况？

角度二：您会给团队业绩制定较高的业绩目标吗？

角度三：什么样的工作态度是您最欣赏的？

优势长处：直属领导有哪些独特的个人优势？

角度一：作为团队负责人，您最突出的个人优势是什么？（如第一次没有回答，可以接着说：不可能，您能做到这个位置肯定有您独特的优势，麻烦您和我分享一点就可以）

角度二：过往工作中，下属在哪方面更需要您的帮助？

角度三：有没有哪一项能力，是您这些年一直在持续关注的？

成长背景：直属领导如何走向现在的管理岗位？

角度一：您愿意和我分享，您是如何走向管理岗的吗？

角度二：这个过程中，您遇到的最大的挑战是什么？ （从请教的角度发问）

角度三：如果能给我一条建议，您认为是什么？

### 4、优势自述+表达意愿

沟通完之后，面试官没有了解到我们自身对于这个职位上的优势时，应主动出击（阐述自身对于岗位优势以及专业技能）

开头：贵公司的情况我大致都了解了，有关贵公司的这个职位（具体到某个点），我有一些自己的想法，想争取您 2 分钟的时间，和您再交流一下。

中间：因为之前我从事 XX 的工作，在 XX 方面沉淀了 XX 能力和 XX 能力（之前的工作成果、个人优势及专业技能），我的理解是，这部分的经历和成果能够迁移使用在贵公司的这个岗位里，不知道我这样理解是否准确呢？

结尾：感谢您给我这次机会，我对公司 XX 和 XX 真的很感兴趣，也希望自己有幸能够加入公司。

# 五、常见面试流程和面试官关注维度

不同职级岗位的常见面试流程和关注维度：

*   初级职位（例如实习生、初级工程师，通常2-3轮面试）：

*   HR面（求职动机+通用面试）

*   部门经理面试（团队经验+通用面试）

*   中级职位（例如中级工程师、项目leader，通常3-4轮面试）：

*   HR面（求职动机+通用面试）

*   团队领导面试（团队经验+通用面试）

*   技术面试或项目经理评估（专业经验+通用面试）

*   高级职位（例如高级工程师、经理、总监，通常4-6轮面试）：

*   HR面（求职动机+通用面试）

*   团队领导面试（团队经验+通用面试）

*   技术面试（专业经验+通用面试）

*   案例面试或领导力面试（问题解决+领导力）

*   高管级职位（例如副总裁、总经理、HRVP，通常6轮以上）：

*   HR初筛（求职动机+通用面试）

*   部门领导、战略面试（领导力+战略思维）

*   领导力面试（领导力+业务视野）

*   公司高管的面试（战略+愿景共鸣）

以上，仅为个人经验输出，不代表官方标准答案，仅供参考。

且面试永远没有标准答案，面试官主要就是通过应聘者在不同场景下的回答思路，判断候选人的综合能力。
