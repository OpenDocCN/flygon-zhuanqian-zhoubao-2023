# 一.站在用户的视角去设计教程

## 1.1 市面上的教程都是什么样的

1.视频课程（普遍内容很长，过渡性话术多）

2.飞书文档（纵向视角）

3.训练营微信群聊天记录（纵向视角）

4.PPT（纵向视角）

最开始接触到知识付费的时候，看到这么多的干货，非常激动，经常听课听得像打鸡血一样，特别带劲。

生财的内容更有价值。

按道理来说，这么多有价值的内容，静下心去学，

1遍，2遍，3遍甚至十几遍，几十遍去看，去照着做。

大概率是能拿到结果的。

但是，28定律注定了，其实还是只有20%的人能拿到结果。

到底什么样的内容形式更容易被一个初学者接受？

流程很烂，总是一步错了就要全部重来。

再复杂的操作，下面也只有几个或者十几个sheet。

## 1.4 从学生的学习路径得到的启发

刚刚写的时候带5岁的儿子去上了积木课，

回想起他学拼音，学写字，每次都是只搞一页，

毫不夸张，就是求着他去学。

上学的时候，每次也都是一节课一节课的学，

再写课后作业，

再考试，

一个知识点是被反复干了无数次，

才去上考场的。

如果他还没读完幼儿园，小学，初中，高中，大学。

我就直接丢给他一堆语数外，理化生，

让他去上大学的选修课。

有问题随时来问。

这么荒诞的事情，没有人会去做。

但是，做项目，做电商，

实际上，很多人幼儿园的阶段都没有跑完。

本身不在一个起跑线，是没法一起跑的。

## 1.5 多一些必做题，少一些选择题

两年前，我写过一篇文章，讲讲我加入过的付费社群。

直到今天来看，这些社群都是非常有价值的，当初的大部分选择并没有错。

但是社群好，内容好，并不代表我能拿到结果。

再没有续费的时候，有的社群问我为什么没续费了。

让我提一下意见。

我说，我希望，能够多一些必做题，少一些选择题。

内容太多了，不知道怎么选。

也许是希望去得到指令式的培训，选什么项目，做什么，而不是自己选。

就像员工一样。

但是这样的交付太重了，而且大概率吃力不讨好，甚至挨骂，你替别人选了，

没有拿到结果就只能社群承担，而不是学员承担。

基本上也没见过这种形式。

# 二.站在普通人的立场去设计流程

## 2.1 普通人的定义：都不会，执行差，有点懒，想搞钱

前面，在评论区看到圈友质疑过某个被安利的项目，可能因为门槛高，觉得不适合普通人。

于是，就有了这样一个问题，到底什么是普通人？

我理解的普通人定义是：

都不会，有点懒，执行差，想搞钱。

我理解这种心态，因为我就是这样的，我想成千上万的人也是如此

## 2.2 普通人去社群参加知识付费到底是为了解决什么问题

已经被很多人总结过无数次的经验，

执行力是做成一个项目极其重要的因素。

那有没有可能，一个用户过来，他以为的是找你学赚钱，

但本质上，他进来就是为了你帮他给他解决执行力的问题。

如果他本身就是一个极其有执行力，又勤奋好学，又舍得花钱链接的人。

那他为什么会跑不出来？为什么要找你付费？

用罗翔老师的话来说: 逻辑它不自洽。
